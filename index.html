مستندات سامانه تحلیل فرکانس

×

معرفی کلی

این مستندات به توضیح عملکرد و APIهای سامانه شما می‌پردازد. این سامانه شامل مدیریت حساب کاربری و عملیات تحلیل است و برای استفاده در کاربردهای مختلف طراحی شده است.

بخش‌هایی از API (API Endpoints)

  • POST /api/accounts/: ایجاد یا ثبت‌نام کاربر جدید.
  • POST /api/accounts/login/: ورود کاربران.
  • POST /api/accounts/logout/: خروج کاربران.
  • GET /api/operations/: دریافت لیست عملیات موجود.
  • POST /api/operations/: ایجاد عملیات جدید (مثلاً تحلیل یا محاسبه).
  • GET /api/operations/{id}/: دریافت جزئیات عملیات مشخص با شناسه id.

مسیرهای صفحات وب (Web Pages)

  • GET /accounts/login/: صفحه ورود کاربران.
  • GET /accounts/profile/: صفحه داشبورد پس از ورود کاربران.
  • GET /operations/new/: فرم ایجاد عملیات جدید.
  • GET /operations/history/: نمایش تاریخچه عملیات کاربر.
  • GET /operations/detail/{pk}/: مشاهده جزئیات عملیات مشخص با شناسه pk.
  • GET /operations/download/{pk}/: دانلود نتیجه عملیات با شناسه pk.

مستندات سامانه تیکتینگ و نظرسنجی

×

معرفی کلی

این مستندات، ساختار و APIهای سامانه تیکتینگ و نظرسنجی دانشگاه صنعتی بیرجند را تشریح می‌کند. این سیستم به منظور بهبود فرآیندهای ارتباطی دانشجویان و اساتید با واحدهای مختلف دانشگاه طراحی شده است.

معماری سیستم

سیستم بر پایه Django REST Framework ساخته شده و از Django Channels برای ارتباطات بلادرنگ (Real-time) استفاده می‌کند. پایگاه داده MySQL برای ذخیره‌سازی اطلاعات به کار رفته است.

بخش‌هایی از API

  • POST /api/tickets: ایجاد یک تیکت جدید.
  • GET /api/tickets: دریافت لیست تیکت‌ها (با قابلیت فیلتر).
  • GET /api/tickets/{id}: دریافت جزئیات یک تیکت خاص.
  • PUT /api/tickets/{id}: به‌روزرسانی یک تیکت.
  • POST /api/surveys: ایجاد یک نظرسنجی جدید.
  • POST /api/surveys/{id}/responses: ثبت پاسخ به یک نظرسنجی.
